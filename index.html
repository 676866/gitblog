Everyday Git

What is Git

Git is a distributed version control system. A version control system helps track changes in source code during software development. It allows multiple developers to collaborate efficiently.

Other version control systems include SVN, Mercurial, and Perforce. This blog focuses on Git and GitHub.

Git Documentation

Installing Git

Check if Git is installed:

git --version

To install Git:

  • Download Git for all platforms
  • Follow the installation instructions on your OS.

Configuring Git

Set your name and email address:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Get Started

Create a folder for your project and an index.html file. You can use any code editor like VS Code ,Webstorm or Atom (code editor of your choice).

Initializing a Git Repository

A Git repository stores your code and version history.

  • first check if you have initialized git or not
    • git status

    git init

    Adding Files to the Staging Area

    The staging area is where changes are prepared before committing.

    git add filename
    git add .

    Tracked files are already in Gitâ€™s version history; untracked files are new and not yet added.

    Committing Changes

    To save changes:

    git commit -m "Add meaningful message"

    View commits:

    git log

    Return to a previous commit:

    git checkout [commit-hash]

    Branches

    Branches let you work on different features or fixes separately. The main branch is usually called main.

    Creating a Branch

    git branch newBranch

    Viewing All Branches

    git branch

    The current branch is marked with *.

    Switching Active Branches

    git checkout branch-name

    Merging a Branch

    First, switch to the branch you want to merge into (usually main):

    git checkout main
    git merge new-feature

    Deleting a Branch

    git branch -d branch-name
